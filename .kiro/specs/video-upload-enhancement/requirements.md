# Requirements Document

## Introduction

Це специфікація для покращення функціональності завантаження відео на сторінці "Спробуйте безкоштовно" KickAI Judge. Система повинна дозволяти користувачам завантажувати відео файли MP4 через drag-and-drop інтерфейс або вибір файлів, з подальшим відображенням відео плеєра для перегляду та аналізу.

## Glossary

- **Video Upload System**: Система завантаження відео файлів на веб-платформу
- **Drag-and-Drop Interface**: Інтерфейс, що дозволяє перетягувати файли з файлової системи в браузер
- **Video Player**: Компонент для відтворення завантажених відео файлів
- **File Validation**: Процес перевірки типу, розміру та формату файлу
- **Upload Progress**: Індикатор прогресу завантаження файлу
- **Video Preview**: Попередній перегляд завантаженого відео

## Requirements

### Requirement 1

**User Story:** Як користувач, я хочу завантажити відео файл MP4 через drag-and-drop, щоб швидко та зручно додати відео для аналізу.

#### Acceptance Criteria

1. WHEN користувач перетягує MP4 файл в зону завантаження THEN система SHALL прийняти файл та почати процес завантаження
2. WHEN користувач перетягує файл неправильного формату THEN система SHALL відхилити файл та показати повідомлення про помилку
3. WHEN файл успішно завантажено THEN система SHALL відобразити попередній перегляд відео
4. WHEN користувач перетягує файл розміром більше 2GB THEN система SHALL відхилити файл та показати повідомлення про перевищення розміру
5. WHILE процес завантаження триває THEN система SHALL відображати індикатор прогресу

### Requirement 2

**User Story:** Як користувач, я хочу вибрати відео файл через файловий браузер, щоб мати альтернативний спосіб завантаження відео.

#### Acceptance Criteria

1. WHEN користувач натискає кнопку "Вибрати файл" THEN система SHALL відкрити діалог вибору файлів
2. WHEN користувач вибирає MP4 файл через діалог THEN система SHALL почати завантаження файлу
3. WHEN користувач вибирає файл неправильного формату THEN система SHALL показати повідомлення про помилку
4. WHEN користувач скасовує вибір файлу THEN система SHALL залишити поточний стан без змін

### Requirement 3

**User Story:** Як користувач, я хочу бачити попередній перегляд завантаженого відео, щоб переконатися що завантажено правильний файл.

#### Acceptance Criteria

1. WHEN відео успішно завантажено THEN система SHALL відобразити відео плеєр з превью кадром
2. WHEN користувач натискає кнопку відтворення THEN система SHALL почати відтворення відео
3. WHEN користувач натискає кнопку паузи THEN система SHALL призупинити відтворення відео
4. WHEN відео відтворюється THEN система SHALL показувати поточний час та загальну тривалість
5. WHEN користувач переміщує повзунок часу THEN система SHALL перемотати відео до вибраної позиції

### Requirement 4

**User Story:** Як користувач, я хочу бачити прогрес завантаження файлу, щоб знати коли процес буде завершено.

#### Acceptance Criteria

1. WHEN починається завантаження файлу THEN система SHALL показати індикатор прогресу з відсотками
2. WHILE файл завантажується THEN система SHALL оновлювати відсоток завершення в реальному часі
3. WHEN завантаження завершено успішно THEN система SHALL приховати індикатор прогресу та показати відео плеєр
4. WHEN виникає помилка під час завантаження THEN система SHALL показати повідомлення про помилку та дозволити повторну спробу

### Requirement 5

**User Story:** Як користувач, я хочу отримувати зрозумілі повідомлення про помилки, щоб знати що пішло не так та як це виправити.

#### Acceptance Criteria

1. WHEN користувач завантажує файл неправильного формату THEN система SHALL показати повідомлення "Підтримуються тільки MP4 файли"
2. WHEN користувач завантажує файл розміром більше 2GB THEN система SHALL показати повідомлення "Розмір файлу не повинен перевищувати 2GB"
3. WHEN виникає мережева помилка THEN система SHALL показати повідомлення "Помилка завантаження. Спробуйте ще раз"
4. WHEN файл пошкоджений THEN система SHALL показати повідомлення "Файл пошкоджений або не може бути відтворений"

### Requirement 6

**User Story:** Як користувач, я хочу мати можливість замінити завантажене відео, щоб проаналізувати інший поєдинок.

#### Acceptance Criteria

1. WHEN відео вже завантажено THEN система SHALL показувати кнопку "Завантажити інше відео"
2. WHEN користувач натискає "Завантажити інше відео" THEN система SHALL очистити поточне відео та повернутися до стану завантаження
3. WHEN користувач завантажує нове відео THEN система SHALL замінити попереднє відео новим
4. WHEN користувач скасовує заміну відео THEN система SHALL залишити поточне відео без змін
